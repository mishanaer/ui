# Source Basis

This skill synthesizes three Russian interface writing traditions:

## Ozon

Useful for component formats and UX writing pedagogy.

Key ideas:

- UX texts help the user move through the flow and improve the user experience.
- Interface copy differs from marketing copy: marketing attracts, interface copy helps.
- Text should account for format, context, tone, and relationship with other UI elements.
- Headings and subheadings should be meaningful units because users scan screens.
- Buttons are the user's action or acknowledgement.
- Alerts, snackbars, tooltips, modals, onboarding, badges, inputs, empty states, errors, and success screens each have distinct jobs.
- Important information goes first.
- Use consistency instead of synonym variety.

Primary public sources:

- Ozon Tech Habr course "Как писать просто, понятно и для интерфейсов: лучшие практики UX-редакторов Ozon" — https://habr.com/ru/companies/ozontech/articles/821383/
- Figma guide mirror/entry point — https://s.ozon.ru/V7186r5

## Raiffeisen

Useful for strict meaning, tone hygiene, and interface syntax.

Key ideas from the user-provided "Текст в интерфейсе" guide:

- Meaning: understandable to a novice, no ambiguity, explain strange things, concrete not abstract, no false promises, sequential.
- Tone: person to person, no bureaucracy, no empty words, no judging feelings, careful with personal traits, no overacting, no fake care, no blame, no excess joy, do not center the brand, gender neutral.
- Interface: homogeneous wording, agreement between elements, elements work without context, heading reads with button, text does not repeat element function, no hanging prepositions, punctuation is consistent.
- Good interface text sounds natural, reads through the whole screen, cares about feelings without assuming them, and checks every mark.

## T-Bank

Useful for a compact product-system policy.

Key ideas:

- General principles: useful, clear, simple.
- Tone: neutral and on equal terms with the user.
- Text should exist only when it adds concentrated value.
- If the interface can be improved so the user does not need to read, prefer improving the interface.
- Component-level writing matters: headings, buttons, alerts, hints, legal text, errors, dates, currencies, and typography need separate treatment.

Primary public source: T-Bank design portal, "Текст" and "Редполитика интерфейсов" sections — https://design.tbank.ru/

## How To Use The Synthesis

When sources conflict, use this priority:

1. User safety, money, legal accuracy, and product truth.
2. Clarity for a first-time user.
3. Component syntax and flow consistency.
4. Calm tone.
5. Brevity.
6. Brand expressiveness.

Do not present this skill as an official Ozon, Raiffeisen, or T-Bank policy. It is a universal working policy derived from their publicly available or user-provided materials.
